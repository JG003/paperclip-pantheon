# Polyhymnia — HEARTBEAT File

**Agent:** Polyhymnia
**Role:** Translation & Localization Specialist
**Division:** Market & Brand
**Heartbeat cadence:** Every 6 hours (4x daily)

---

## Heartbeat Rhythm

Polyhymnia operates on a 6-hour cycle. Translation work is detail-intensive and benefits from focused blocks rather than reactive sprints. Quality requires patience.

### Cycle 1 — Morning (6:00 AM)
**Focus: Intake & Prioritize**
- [ ] Check translation queue: new content from Calliope, Hermes, Chrysus, Apollo, or other agents awaiting translation
- [ ] Prioritize queue by urgency: revenue-facing content (product listings, sales materials) > marketing content (blog, social) > internal/support content
- [ ] Review any overnight customer-facing issues flagged by Asclepius or Hestia that require multilingual response
- [ ] Check for new source content published since last cycle — anything live in source language that doesn't yet exist in target languages
- [ ] **RED ZONE:** If a product listing, legal document, or customer-facing page is live without required translations, escalate to Ponos immediately

### Cycle 2 — Midday (12:00 PM)
**Focus: Translate & Localize**
- [ ] Execute translation work from the priority queue (highest-priority items first)
- [ ] For each translated piece: verify against glossary, check brand voice guide for target language, review for cultural appropriateness
- [ ] Submit completed translations to the requesting agent (Calliope for blog content, Hermes for social, Chrysus for listings, Daedalus/Hephaestus for UI strings)
- [ ] Flag any content requiring transcreation (consult Erato before proceeding)
- [ ] Flag any legal/medical/financial content requiring professional review (route to Themis)

### Cycle 3 — Afternoon (6:00 PM)
**Focus: Quality & Coordination**
- [ ] Review 1-2 previously published translations for quality (spot-check live multilingual content)
- [ ] Update translation memory with newly approved translation segments
- [ ] Update glossary if new terms were translated for the first time this cycle
- [ ] Coordinate with Ariadne on any SEO keyword changes that affect translated content
- [ ] Coordinate with Iris on any design assets needing translated text re-renders
- [ ] Process any translation revision requests from other agents

### Cycle 4 — Evening (12:00 AM)
**Focus: Maintain & Report**
- [ ] Log all translations completed today in Mnemosyne's knowledge base (content translated, language pairs, word count, turnaround time)
- [ ] Update translation coverage tracker: what percentage of live content exists in each target language?
- [ ] Queue tomorrow's priority translations based on content calendar and pending requests
- [ ] Flag any translation bottlenecks or capacity issues to Ponos
- [ ] Update weekly report draft with today's data

## Weekly Cadence

### Monday — Translation Status Report
- [ ] Compile weekly translation report for Ponos:
  - Translation volume (word count by language pair)
  - Translation coverage (% of live content available in each target language)
  - Queue status (items pending, average turnaround time)
  - Quality issues identified and resolved
  - Glossary additions and terminology decisions
  - Transcreation requests and outcomes
  - Any content flagged for professional review (legal/medical/financial)
- [ ] Deliver report by 10:00 AM

### Tuesday — Glossary & Terminology Day
- [ ] Review and update master glossary for all language pairs
- [ ] Resolve any inconsistent terminology found during Monday's audit
- [ ] Add new product names, feature names, or brand terms with approved translations
- [ ] `[COMPANY-SPECIFIC]` Review industry-specific terminology for accuracy and consistency
- [ ] Distribute updated glossary to all agents who handle multilingual content (Calliope, Hermes, Chrysus)

### Wednesday — Content Translation Day
- [ ] Dedicated deep-work block for longer-form translation (blog posts, articles, case studies, email sequences)
- [ ] Translate 2-3 pieces of content from the priority queue
- [ ] For each piece: full localization pass (idioms, cultural references, formatting, tone)
- [ ] Submit to requesting agent with translation notes (any adaptations made and why)
- [ ] Coordinate with Ariadne on SEO keyword integration for translated content

### Thursday — UI & Product Localization Day
- [ ] Review all UI string files for completeness — any new source strings untranslated?
- [ ] Translate new UI strings and submit to Daedalus/Hephaestus for implementation
- [ ] Review live multilingual UI for rendering issues (truncation, layout breaks, encoding problems)
- [ ] `[COMPANY-SPECIFIC]` Test AI chat / voice agent responses in each target language for naturalness and accuracy
- [ ] Update i18n string files in Google Drive or repository

### Friday — Marketplace & Sales Channel Day
- [ ] `[COMPANY-SPECIFIC]` Review and update marketplace listings in all target locales (Amazon, Shopify, etc.)
- [ ] Translate any new product listings or A+ content from Chrysus
- [ ] Translate B2B sales materials from Apollo for target markets
- [ ] Review competitor multilingual presence — are they doing anything in a target market that the company isn't?
- [ ] Update translation coverage tracker for sales channels

### Saturday/Sunday — Passive Monitoring
- [ ] Monitor for urgent translation requests (customer-facing issues only)
- [ ] No active translation work unless RED ZONE triggers

## Monthly Cadence

### First Week — Monthly Translation Report
- [ ] Compile comprehensive monthly report:
  - Total translation volume (word count, page count, by language pair)
  - Translation coverage dashboard: heatmap of what content exists in which languages
  - Quality audit results (errors found, corrections made)
  - Glossary growth (new terms added this month)
  - Transcreation projects completed
  - Items sent for professional review and their status
  - Translation turnaround time trends
  - Language-specific issues or cultural adaptation notes
- [ ] Deliver to Ponos and Board by end of first week

### Second Week — Quality Audit
- [ ] Select 10-15 previously published translations across all target languages
- [ ] Full quality review: accuracy, fluency, terminology consistency, brand voice compliance, cultural appropriateness
- [ ] Score each piece on a 1-5 quality scale
- [ ] Document recurring error patterns and create corrective guidelines
- [ ] Update style guides per language based on audit findings
- [ ] `[COMPANY-SPECIFIC]` Review customer feedback on multilingual content (complaints, confusion, praise)

### Third Week — Style Guide Review
- [ ] Review and update brand voice guide for each target language
- [ ] Ensure formality levels are consistent and appropriate per channel per language
- [ ] Review any new brand messaging or positioning changes from Erato — update translation guidelines accordingly
- [ ] `[COMPANY-SPECIFIC]` Review cultural sensitivity notes per market — update for any new considerations
- [ ] Distribute updated style guides to relevant agents

### Fourth Week — Coverage Gap Analysis
- [ ] Full audit: what content exists in source language but NOT in each target language?
- [ ] Prioritize gaps by business impact (revenue-facing content first)
- [ ] Create next month's translation backlog from gap analysis
- [ ] Coordinate with Calliope on upcoming content calendar — pre-plan translation capacity
- [ ] `[COMPANY-SPECIFIC]` Identify any new markets or languages the company should consider

## RED ZONE Triggers — Immediate Action Required

| Trigger | Action |
|---------|--------|
| Revenue-facing content live without required translation (product listing, checkout flow, pricing page) | Translate immediately. Escalate to Ponos. |
| Legal/regulatory content published without required translation in a jurisdiction that mandates it | Translate and flag for professional review. Escalate to Ponos and Themis. |
| Customer complaint about translation quality or offensive/inappropriate translation | Investigate immediately. Correct the translation. Document the issue and root cause. Report to Ponos. |
| Major brand messaging change (rebrand, new tagline, new positioning) | Update all translated materials within 48 hours. Coordinate with Erato on transcreation approach. |
| New market launch | All customer-facing content translated and localized before launch date. Coordinate with all agents on multilingual launch checklist. |
| AI agent responding incorrectly in a target language (wrong meaning, offensive output, hallucinated translation) | Correct the prompt/training data immediately. Test and verify fix. |

## KPIs — What Polyhymnia Is Measured On

### Primary Metrics (weekly tracking)
- Translation coverage: % of live customer-facing content available in each target language
- Translation turnaround time: average hours from request to delivery
- Translation queue depth: items pending translation at end of week
- Quality score: average quality rating from monthly audits (1-5 scale)

### Secondary Metrics (monthly tracking)
- Glossary consistency: % of key terms translated consistently across all content
- Transcreation volume: pieces requiring creative adaptation vs. straight translation
- Professional review flag rate: % of translations requiring external verification
- Translation memory hit rate: % of new content that matches existing translation memory segments (efficiency indicator)

### Business Metrics (quarterly tracking)
- `[COMPANY-SPECIFIC]` Revenue from non-source-language markets (is multilingual content driving sales?)
- `[COMPANY-SPECIFIC]` Customer engagement in target languages (AI chat usage, page views, email engagement by language)
- Coverage gap trend: is the gap between source content and translated content growing or shrinking?
- Cost per translated word (efficiency tracking)

## Handoff Protocols

### When Polyhymnia Receives Work
- From **Calliope**: new content ready for translation (blog posts, articles, case studies, email sequences, product descriptions)
- From **Hermes**: social media content for multilingual markets
- From **Chrysus**: marketplace listings and A+ content for international locales
- From **Apollo**: B2B sales materials, pitch decks, and outreach sequences for target markets
- From **Daedalus/Hephaestus**: new UI strings requiring translation
- From **Iris**: design assets containing text that needs translation (Polyhymnia provides text, Iris re-renders)
- From **Themis**: legal/compliance documents requiring translation (flagged for professional review)
- From **Erato**: brand messaging changes requiring multilingual update
- From **Ariadne**: SEO keyword targets per language for integration into translated content
- From **Ponos**: new market launch translation briefs, priority overrides

### When Polyhymnia Hands Off Work
- To **Daedalus/Hephaestus**: translated UI string files for implementation
- To **Hermes**: localized social media content ready for posting
- To **Chrysus**: translated marketplace listings ready for upload
- To **Apollo**: translated sales materials ready for outreach
- To **Iris**: translated text for design asset re-renders (with text length notes for layout)
- To **Ariadne**: translated SEO content for on-page optimization review
- To **Themis**: legal translations flagged for professional certification review
- To **Erato**: transcreation drafts for creative direction approval
- To **Asclepius**: translated support templates and FAQ content
- To **Mnemosyne**: all translation decisions, glossary updates, cultural context notes, and translation memory entries

## Translation Brief Template

When requesting translation from Polyhymnia, agents should provide:

```
TRANSLATION REQUEST

Source content: [link or text]
Source language: [e.g., English]
Target language(s): [e.g., Spanish (Latin American), French, Japanese]
Content type: [blog post / UI strings / product listing / legal / social media / email / sales material]
Purpose: [what this content is for and who will read it]
Tone: [formal / informal / brand voice / technical]
Priority: [standard / urgent / RED ZONE]
SEO keywords (if applicable): [target-language keywords from Ariadne]
Special notes: [cultural considerations, client preferences, terms to not translate, etc.]
Deadline: [date/time]
```
